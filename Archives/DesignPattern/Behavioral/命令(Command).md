# 命令(Command)

## 特点

- 将请求参数包装成一个特定的命令对象. 并可以使用待执行队列由处理者逐一处理

## 结构

- 发送者(Sender/Invoker)
  - 触发对应的事件. 并生成对应的命令
- 命令接口(Interface Command)
  - 基本的命令结构的定义 如: type. info.
- 具体命令(ConcreteCommand) 继承命令接口
  - 根据发送者的数据包装成特定的命令
- 接收者(Receiver)
  - 接受各式命令, 并进行对应的处理
- 客户端(Client)
  - 包含接收者的实例. 和 待处理队列等

## 使用场景

- 通过操作来参数化对象
  - 如: execute('COPY')
- 操作放入队列,远程操作,操作执行等
- 实现操作回滚.
  - undo 操作. 可以记录每次命令的数据

## 结论

### 优点

- 单一职责: 解耦触发和执行
- 开闭原则: 不修改已有客户端代码的情况下创建新命令
- 可以实现撤销和恢复功能
- 实现操作的延迟执行
- 将多个命令组成复杂的命令

### 缺点

- 代码复杂度上升. 在发送者和执行者 加入了一个新层次
