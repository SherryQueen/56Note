# 代理(Proxy)

## 优点

- 客户端通过代理对象来访问实际对象,而非直接操作实际对象

## 特点

- 客户端的所有操作只与代理对象有关, 与实际对象无关.

## 结构

- 代理对象(Proxy)
  - 暴露一定的方法与客户端交互,并根据对应的方法操作实际对象

## 使用场景

- 保护实际对象的操作可控
- 不便于暴露实际对象的环境

## 代码示例

```Typescript
const obj = {}

const proxy = new Proxy(obj, {...})

```

## 区别

- 外观模式: 为一系列接口封装更高级的对外接口, 更易用(如: 合并复杂的一些操作)
- 装饰器模式: 增强方法/类本身的能力, 客户端依然直接操作实际对象
- 代理模式: 通过代理对象对实例进行访问.
