# Redis

- 基于内存的内存数据库，常用于缓存数据

## 特点

- 基于内存，避免 IO 读写影响数据交互，访问速度块
- 支持 key-value, list, hash, set 等多种数据结构
- 支持数据备份 主从模式
- 支持事务
- 支持本地持久化

## 优势

- 性能高。 读写速度极快
- 数据类型丰富。
- 操作为原子。 多原子操作合并为单个事务
- 支持发布/订阅 通知 key 过期的等特性

## 实现大致特点

- 基于内存实现，减少不必要的 IO 操作
- 数据结构高效
  - 压缩空间 更有效率的内存空间分配与管理
  - 双向链表 更灵活的支持队列/数组等数据结构
  - 跳跃表 多级索引 更灵活更高的查找效率
  - 主线程处理逻辑，保证原子性，避免频繁的上下文切换开销。 多线程提供 IO/事件处理等其他功能
  - IO 复用 避免多 socket 链接导致 更多的线程创建/销毁开销

## 常见问题

- 缓存雪崩
  - 现象: 缓存出现意外，导致请求进入数据库查询阶段。导致数据库故障。
  - 方案
    - Redis 高可用 支持主从+哨兵 避免全面崩溃
    - MySQL 支持缓存，对数据库查询添加限流与降级。避免数据库宕机
    - Redis 持久化，重启时快速回复数据
- 缓存穿透
  - 现象: 查询的数据不在缓存中，导致缓存失效
  - 方案:
    - 每次查询后更新至缓存+过期时间
- 缓存击穿
  - 现象: 热门缓存突然失效，导致缓存击穿
  - 方案:
    - 热门缓存永不过期
    - 支持查询互斥锁，后续查询等待第一次查询更新缓存后继续查询
