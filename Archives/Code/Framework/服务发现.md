# 服务发现

## 概念

- 服务提供者 (提供对外服务)
- 服务消费者 (访问服务提供者处理一些事务)
  - 一个服务既可以是服务消费者, 也可以是服务提供者
  - 当服务依赖于其他服务的提供的服务时，即为服务依赖
- 服务中介 (提供多个服务，每个服务后面可能有多个服务提供者)

  - 注册: 记录哪个服务提供者作为服务的后续一员
  - 查找: 根据对应的服务名称得到 服务提供者 提供的服务
  - 在服务提供者挂掉时，要取消对应的注册
  - 新的服务提供者加入时，能安排服务消费者向新服务要求服务

## 问题

- 服务提供者意外挂掉

  - 提供者主动取消注册
  - 服务中介 通过保活机制 确认 服务提供者还在提供服务

- 服务列表 变动如何通知消费者
  - 轮询 (服务多时消耗大)
    - 添加版本号，当服务变动时即递增版本号，通过版本号的变动从而得知服务列表是否改变
  - pubsub (每个发布/订阅占用线程和 redis 链接)
    - 全局广播版本号变动

## 应用场景

- 对外提供的 HTTP 服务
- 其他各类可对外提供的服务
  - MySQL
  - UDP
  - 等

## 延伸

- 服务配置重载
  - 集成配置重载功能,但配置发生变更,服务中介会告知相关服务变更配置信息
- 服务管理后台
  - 监控/查看 所有服务和消费者的消息。根据对应的服务依赖,可查看到完整的服务依赖树
