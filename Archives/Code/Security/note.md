安全漏洞

- 输入输出验证
  - SQL 注入
    - 直接根据用户传入的字符串拼接 SQL 字符串 | 未过滤字段
    - 危害
      - 脱库
      - 修改库数据
  - 越权
    - 垂直越权 | 后端未做校验
    - 水平越权 | 修改请求的用户 id， mock 他人发起的请求
    - 未授权访问 | 未做权限校验
    - 危害
      - 普通用户有了高权限 API 的操作权限
      - 可以获取到他人的数据信息
  - 文件上传
    - 上传可执行文件
    - 替换现有静态资源文件
    - WebShell 网页可执行命令行
    - 服务端可能解析对应的文件
    - 预防：
      - 后端二次校验
      - 后缀校验
      - 文件重命名
      - 文件剥夺可执行权限
  - 文件读取
    - 直接根据文件路径读取源码
    - 预防：
      - 下载地址存入数据库或 oss
      - 文件 id 使用随机值
      - 文件下载鉴权
      - 记录下载日志
  - XSS
    - 前端输入 js 代码， 在他人打开页面时，执行输入的 js 代码 (客户端信任服务端)
    - 预防：
      - cookie 只读
      - Html 字符转译
  - CSRF
    - 登陆 A 生成 Cookie, 未登出 A 的情况下登陆 B, B中添加诱导链接, 如 img src, get请求等 访问A
    - 预防:
      - referer 验证
      - token 验证
        - 危险操作, 添加随机 token 校验判断请求是否合法
  - 调试泄露
    - 泄露后端调试信息， 可能包含敏感数据
  - 明文传输
    - Token in url
  - url 跳转
    - 重定向到其他网站， 获取用户凭证
  - 错误提示不当
    - 用户不存在/用户或密码错误
  - 短信轰炸
    - 相同手机号验证码发送次数
  - 爆破漏洞
    - 其余平台获取到账户/密码信息，通过登陆接口实现爆破登陆
  - Remote rejection/Remote Code execution
    - 服务端根据传入的部分字段进行命令拼接并执行

安全原则：

- 输入不可靠
  - 直接输入
  - 数据来源于间接输入
- 输入最小化
- 返回信息最小化
- 输入范围最小化
- 权限最小化

通用原则：

- 不依赖安全配置
- 安全控制错误落实在最后
  - 输出编码在提交时检查, 统一编码
  - 统一防御手段， 基于框架或 AOP
- 简单原则
  - 逻辑尽量简单，清晰
  - 减少编码冗余，提高复用率
